TeenPle은 기능 구현에서 끝난 프로젝트가 아니라, 모바일 앱 출시와 백엔드 운영까지 이어진 서비스입니다. 사용자 화면, 인증·게시판·채팅 API, AWS 운영 환경까지 한 흐름으로 연결했습니다.

ARCHITECTURE · OPERATION

출시 이후를 고려한
AWS 운영 구조

모바일 앱 요청은 Route 53과 ALB를 거쳐 Spring Boot 서버로 전달됩니다. DB와 캐시는 Private Subnet에 두고, 이미지 저장·유해 이미지 검사·로그 수집까지 운영에 필요한 흐름을 함께 설계했습니다.

운영 안정성

ALB, EC2, RDS, Redis를 분리해 서비스 계층과 데이터 계층을 나눴습니다.

콘텐츠 안전성

이미지 업로드는 S3와 Rekognition 흐름으로 검수 가능하게 구성했습니다.

배포 자동화

GitHub Actions와 Docker 기반으로 배포 반복 비용을 줄였습니다.

운영 관찰

CloudWatch Logs로 애플리케이션 로그를 확인할 수 있게 했습니다.
