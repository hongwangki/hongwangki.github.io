TeenPle

// OVERVIEW

학생들이 안전하게 소통하는
익명 커뮤니티

TeenPle은 같은 학교 학생들끼리 익명으로 글과 사진을 공유하는 커뮤니티 서비스입니다. Java · Spring Boot로 API 서버를 구현하고, 트래픽 증가에 따라 자동으로 확장되는 AWS 인프라 위에 배포했습니다.

익명 서비스의 특성상 부적절한 이미지 업로드가 큰 리스크였기에, Amazon Rekognition으로 유해 이미지를 자동 검열한 뒤 통과한 이미지만 S3에 저장하는 파이프라인을 구성했습니다.

Tech Stack
Java Spring Boot MySQL Redis AWS EC2 ALB · ASG S3 Rekognition CloudWatch
Keywords
#멀티_AZ_고가용성 #오토스케일링 #이미지_검열_파이프라인
// SYSTEM ARCHITECTURE

AWS 멀티 AZ 클라우드 인프라

Route 53부터 ALB · Auto Scaling · Private Subnet의 RDS / Redis, 그리고 Rekognition 이미지 검열까지 — 가용성과 확장성을 고려해 설계한 전체 구조입니다. 이미지를 클릭하면 크게 볼 수 있습니다.

클릭하여 확대

TeenPle — AWS Cloud Architecture · Multi-AZ VPC

Route 53 · ALB

API 라우팅 & 로드 밸런싱

Route 53가 API 도메인을 ALB로 라우팅하고, ALB가 멀티 AZ로 요청을 분산합니다.

EC2 · Auto Scaling

오토스케일링 앱 서버

Spring Boot API가 Private App Subnet에서 최대 2대까지 자동 확장됩니다.

Rekognition · S3

유해 이미지 검열

Rekognition으로 이미지를 검사한 뒤, 통과한 이미지만 S3에 저장합니다.

RDS MySQL · Redis

데이터 & 캐시 계층

메인 DB(RDS MySQL)와 인메모리 캐시(ElastiCache Redis)를 Private Data Subnet에 격리 배치했습니다.

CloudWatch Logs

로그 & 지표 수집

애플리케이션 로그와 지표를 CloudWatch Logs로 모아 모니터링합니다.

Systems Manager

안전한 운영 접근

SSH 대신 Session Manager로 EC2에 접근하고, NAT Gateway를 통해 아웃바운드 연결을 유지합니다.
