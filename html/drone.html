PROBLEM

동시 주문 시 데이터 유실

주문이 몰리면 일부가 DB에 저장되지 않는 현상 발생.

CAUSE

트랜잭션 안의 Blocking I/O

@Transactional 내부의 외부 호출이 DB 커넥션을 장시간 점유 → 병목.

SOLUTION

트랜잭션 축소 + 비동기 분리

트랜잭션은 DB 저장만, 외부 통신은 BlockingQueue로 분리해 별도 스레드 병렬 처리.
