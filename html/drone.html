// PROBLEM SOLVING · 한눈에

병목을 추적하고 구조로 해결

PROBLEM

동시 주문 시 데이터 유실

동시 주문이 몰리면 일부 데이터가 DB에 저장되지 않는 현상 발생. 로그 추적으로 병목 지점을 탐색.

CAUSE

트랜잭션 안의 Blocking I/O

@Transactional 내부의 외부 서버 호출이 Blocking I/O로 동작해 DB 커넥션을 오래 점유 → 병목 유발.

SOLUTION

트랜잭션 축소 + 비동기 분리

트랜잭션 범위를 DB 저장으로 축소하고, 외부 통신은 BlockingQueue 기반 비동기 구조로 분리해 별도 스레드에서 병렬 처리.

IMPROVE

Race Condition · 데드락 방지

재고 차감 Race Condition을 비관적 락으로 해결, 상품 ID 기준 락 순서 고정으로 데드락 방지·정합성 확보.
