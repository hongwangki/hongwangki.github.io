01PROBLEM · 문제 상황

동시 주문 시 데이터 유실

동시 주문이 몰리면 일부 데이터가 DB에 저장되지 않는 현상이 발생했습니다. 팀원들과 함께 로그를 추적하며 시스템 병목 지점을 탐색하고 분석했습니다.

02CAUSE · 원인 분석

트랜잭션 안의 Blocking I/O

@Transactional 내부의 외부 서버 호출이 Blocking I/O로 동작하면서, DB 커넥션을 불필요하게 오래 점유해 병목을 유발하고 있었습니다.

03SOLUTION · 해결 방안

트랜잭션 축소 + 비동기 분리

트랜잭션 범위를 DB 저장으로만 축소하고, 외부 통신은 BlockingQueue 기반 비동기 구조로 분리해 별도 스레드에서 병렬 처리하도록 재설계했습니다.

OrderService.java
// 트랜잭션은 DB 저장까지만 — 빠르게 커넥션 반환
@Transactional
public void placeOrder(Order order) {
    orderRepository.save(order);
    deliveryQueue.offer(order);   // 외부 호출은 큐로 위임
}

// 별도 워커 스레드가 비동기로 외부 통신 처리
while (running) {
    Order o = deliveryQueue.take();
    droneClient.dispatch(o);
}
04IMPROVEMENT · 추가 개선

Race Condition & 데드락 방지

재고 차감 시 Race Condition을 해결하기 위해 비관적 락을 적용하고, 데드락을 방지하기 위해 상품 ID 기준으로 락 순서를 고정해 데이터 정합성을 확보했습니다.
