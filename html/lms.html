01PROBLEM · 문제 상황

응답 시간 최대 7~8초

동시 접속자가 늘어날수록 수강신청 페이지 응답 시간이 최대 7~8초까지 치솟아 실제 수강 성공 여부에 영향을 주었습니다. k6 부하 테스트와 로그 분석으로 병목 구간을 식별했습니다.

02CAUSE · 원인 분석

N+1 쿼리와 캐싱 부재

JPA의 Lazy Loading으로 N+1 쿼리가 발생하고, 동일한 강의 · 교수 · 강의계획서 데이터를 반복 조회했습니다. 모든 요청이 DB를 직접 조회하는 구조라 부하가 집중되었습니다.

03SOLUTION · 해결 방안

쿼리 · 캐시 · 직렬화 3단 개선

Q

Query Optimization

Fetch Join으로 연관 엔티티를 단일 쿼리로 조회해 N+1 제거.

C

Caching Strategy

변경 빈도가 낮은 강의 목록에 Redis 캐시 적용, TTL 기반 무효화 전략 설계.

S

Serialization

양방향 연관관계의 순환 참조를 @JsonIgnoreProperties로 해결.

Before2.28s
After0.07s
04IMPROVEMENT · 추가 개선

정원 차감 동시성 제어

정원 차감 시 발생하는 Race Condition을 비관적 락으로 해결하고, 데드락을 방지하기 위해 강의 ID 기준으로 락 순서를 고정해 교착 상태를 막았습니다.
