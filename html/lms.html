PROBLEM SOLVING

수강신청 시간대에 무너지는 응답을 추적했습니다

수강신청은 사용자가 원하는 강의를 확보하는 순간의 서비스입니다. 이 화면에서는 1초의 지연도 단순한 불편이 아니라 수강 실패 가능성으로 이어질 수 있기 때문에, 응답 속도와 정원 차감의 정확성을 함께 고려해야 했습니다.

사용자 관점동시에 접속자가 몰리는 짧은 시간 안에 조회와 신청이 끝나야 합니다.
시스템 관점강의 목록은 빠르게 내려주고, 신청 요청은 중복 차감 없이 끝까지 처리되어야 합니다.
개선 기준로그와 k6 부하 테스트로 병목을 확인한 뒤 쿼리, 캐시, 락 순서로 좁혀갔습니다.
PROBLEM

응답 시간 최대 7~8초

동시 접속자가 늘수록 수강신청 응답이 지연되어 실제 수강 성공 여부에 영향을 줄 수 있는 상태였습니다.

CAUSE

N+1 쿼리와 반복 조회

JPA Lazy Loading으로 N+1 쿼리가 발생했고, 동일 강의·교수·강의계획서 데이터가 요청마다 DB에서 반복 조회됐습니다.

SOLUTION

Fetch Join과 Redis 캐싱

Fetch Join으로 연관 엔티티를 단일 쿼리로 조회하고, 변경 빈도가 낮은 강의 목록은 Redis 캐시에 저장해 DB 직접 조회를 줄였습니다.
