// PROBLEM SOLVING · 한눈에

N+1을 걷어내고 캐시로 떠받치다

PROBLEM

응답 시간 최대 7~8초

동시 접속자가 늘수록 수강신청 응답이 7~8초까지 치솟아 수강 성공에 영향. k6 부하 테스트·로그로 병목 식별.

CAUSE

N+1 쿼리와 캐싱 부재

Lazy Loading으로 N+1 쿼리 발생, 동일 강의·교수 데이터 반복 조회. 모든 요청이 DB 직접 조회 → 부하 집중.

SOLUTION

쿼리 · 캐시 · 직렬화 3단 개선

Fetch Join으로 연관 엔티티를 단일 쿼리로 조회해 N+1 제거, 변경 빈도 낮은 데이터는 Redis 캐싱(TTL 무효화), 순환 참조는 @JsonIgnoreProperties로 해결.

Fetch JoinRedis Cache · TTL@JsonIgnoreProperties
IMPROVE

정원 차감 동시성 제어

정원 차감 Race Condition을 비관적 락으로 해결, 강의 ID 기준 락 순서 고정으로 데드락 방지.
